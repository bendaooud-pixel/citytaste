# Keyword Queue

The file `keyword-queue.json` is the central content planning queue for CityTaste. Each entry represents a target keyword that should become a published page. To add new targets, append an object following the existing schema. The `status` field controls the pipeline: set it to `"queued"` when a keyword is ready for content creation, `"published"` once the page is live and indexed, or `"skip"` if the keyword has been consolidated into another page (add a `notes` field pointing to the pillar it merged into). Keywords with `locale: "fr"` should be written entirely in French; all others default to English.

When picking the next keyword to write, prioritize entries that already have GSC traction (check the `notes` field for ranking data), then move to high-volume `"queued"` items. Use the `suggestedSlug` and `suggestedTitle` as starting points but adjust the title for click-through rate once you see the actual SERP. Replace `{year}` in titles with the current year at publish time. After publishing, update `status` to `"published"` and optionally add a `"publishedDate"` field so you can track content freshness and know when pages need annual refreshes.
